## 3. 冒险处理效率

### 3.1 数据转发

| 转发路径 | 延迟 | 场景 |
|----------|------|------|
| EX/MEM → EX | 0 周期 | 连续 ALU 指令间的 RAW 依赖 |
| MEM/WB → EX | 0 周期 | 隔一条指令的 RAW 依赖 |
| Load-Use | 1 周期 | Load 后立即使用（插入 1 个 bubble） |

### 3.2 分支预测

| 指标 | 值 |
|------|------|
| 预测器类型 | 2-bit 饱和计数器 BHT |
| BHT 大小 | 256 项 × 2 bit |
| 索引方式 | PC[9:2] |
| 初始状态 | WN（弱不跳转） |
| 正确预测惩罚 | 0 周期 |
| 错误预测惩罚 | 2 周期（冲刷 IF/ID） |
| JALR 处理 | 总是冲刷（无间接跳转预测） |

对于循环类分支（如 matmul 三重循环），BHT 训练后预测准确率高，减少控制冒险惩罚。

### 3.3 乘除法

| 操作 | 延迟 | 实现方式 |
|------|------|----------|
| MUL/MULH/MULHSU/MULHU | 2 周期 | 组合逻辑（DSP48E1） + 启动 stall |
| DIV/DIVU/REM/REMU | 34 周期 | 32 周期 restoring division + 启动/完成 |
| 除零 | 1 周期 | 组合旁路 |

## 5. 设计亮点

1. **三路全转发**：EX/MEM/WB 均可转发，覆盖绝大多数数据冒险，仅 Load-Use 需 1 周期 stall
2. **BHT 分支预测**：256 项两位饱和计数器，对循环类分支预测准确率高
3. **乘除法分离设计**：乘法单周期组合逻辑（映射 DSP），除法 32 周期迭代
4. **Harvard 架构**：指令/数据分离，消除取指-访存结构冒险
5. **M-mode 完整 CSR**：支持 ECALL/MRET、Timer 中断、mcycle/minstret 性能计数器
6. **BRAM 存储器**：IMEM/DMEM 推断为 Block RAM，大幅降低 LUT 占用
