| 项目 | 规格 |
|------|------|

| 仿真工具 | Icarus Verilog 12.0 |

| 时钟频率 | 50 MHz（仿真时钟周期 20 ns） |

## 2. 指令级测试性能

| 测试程序 | 周期数 | 指令数（估计） | CPI（估计） | 说明 |
|----------|--------|----------------|-------------|------|
| test_alu | 138 | ~80 | 1.73 | RV32I 算术/逻辑/移位/比较/LUI/AUIPC |
| test_mem | 100 | ~50 | 2.00 | Load/Store 各类型（含 Load-Use stall） |
| test_branch | 116 | ~70 | 1.66 | 分支/跳转含循环（BHT 预测生效） |
| test_muldiv | 318 | ~60 | 5.30 | 含 11 次除法（每次 33+ 周期） |

| matmul (app) | 9368 | ~5000 | 1.87 | 4×4 矩阵乘法 + UART 输出 |

**说明**：
- 理想 CPI = 1.0（每周期退休一条指令）
- CPI > 1.0 的原因：分支 misprediction（2 周期惩罚）、Load-Use stall（1 周期）、除法（33 周期）
- test_muldiv 的高 CPI 主要由除法迭代造成

## 3. 冒险处理效率

### 3.1 数据转发

| 转发路径 | 延迟 | 场景 |
|----------|------|------|
| EX/MEM → EX | 0 周期 | 连续 ALU 指令间的 RAW 依赖 |
| MEM/WB → EX | 0 周期 | 隔一条指令的 RAW 依赖 |
| Load-Use | 1 周期 | Load 后立即使用（插入 1 个 bubble） |

### 3.2 分支预测

| 指标 | 值 |
|------|------|
| 预测器类型 | 2-bit 饱和计数器 BHT |
| BHT 大小 | 256 项 × 2 bit |
| 索引方式 | PC[9:2] |
| 初始状态 | WN（弱不跳转） |
| 正确预测惩罚 | 0 周期 |
| 错误预测惩罚 | 2 周期（冲刷 IF/ID） |
| JALR 处理 | 总是冲刷（无间接跳转预测） |

对于循环类分支（如 matmul 三重循环），BHT 训练后预测准确率高，减少控制冒险惩罚。

### 3.3 乘除法

| 操作 | 延迟 | 实现方式 |
|------|------|----------|
| MUL/MULH/MULHSU/MULHU | 2 周期 | 组合逻辑（DSP48E1） + 启动 stall |
| DIV/DIVU/REM/REMU | 34 周期 | 32 周期 restoring division + 启动/完成 |
| 除零 | 1 周期 | 组合旁路 |

## 5. 设计亮点

1. **三路全转发**：EX/MEM/WB 均可转发，覆盖绝大多数数据冒险，仅 Load-Use 需 1 周期 stall
2. **BHT 分支预测**：256 项两位饱和计数器，对循环类分支预测准确率高
3. **乘除法分离设计**：乘法 1 周期组合逻辑（映射 DSP），除法 32 周期迭代
4. **Harvard 架构**：指令/数据分离，消除取指-访存结构冒险
5. **M-mode 完整 CSR**：支持 ECALL/MRET、Timer 中断、mcycle/minstret 性能计数器
