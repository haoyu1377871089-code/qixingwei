# "七星微"企业命题

## 一、题目名称

基于 RISC-V 的高性能 CPU 设计及 FPGA 验证

## 二、参赛组别

A 组、B 组

## 三、赛题背景与意义

RISC-V 作为开源、精简且可扩展的指令集架构（ISA），已成为处理器设计、学术研究及工业界创新的核心载体。Verilog/VHDL 作为成熟稳定的硬件描述语言，在工业界硬件设计中应用广泛，具备语法严谨、兼容性强、与 FPGA/ASIC 设计流程适配性好等优势，是硬件设计领域的基础核心工具。

本项目主要面向研究生阶段，聚焦"RISC-V 架构+Verilog/VHDL 设计"的核心组合，要求设计一款具备一定性能的 RISC-V CPU 并完成 FPGA 原型验证与实例运行。项目周期为 8 个月，旨在推动研究生深入掌握处理器架构核心原理、Verilog/VHDL 硬件建模方法、FPGA 验证流程及性能优化技术，避免直接复用开源资料完成设计，最终培养硬件系统设计、验证与集成的综合科研能力，为后续学术研究（如异构计算、低功耗设计）或工程应用奠定基础。

## 四、赛题目标

1. 基于 RISC-V 基础整数指令集（RV32I 或 RV64I），采用 Verilog 或 VHDL 设计一款功能完整的 CPU，支持整数运算、数据传输、控制转移等核心指令，预留指令集扩展接口（如 M 扩展、C 扩展）。

2. 实现 CPU 五级流水线架构（取指、译码、执行、访存、写回），解决流水线冒险（数据冒险、结构冒险、控制冒险）问题，保障指令高效并行执行。

3. 完成 CPU 从 Verilog/VHDL 建模、仿真验证到 FPGA 综合实现的全流程开发，确保其能在 FPGA 平台上正确执行 RISC-V 测试程序与应用实例。

4. 引入至少 2 种高性能优化技术（如分支预测、数据旁路、指令预取等），提升 CPU 的指令吞吐量与执行效率。

5. 开发适配该 CPU 的测试程序与简单应用（如矩阵乘法、排序算法），在 FPGA 上完成功能演示与性能量化分析。

## 五、项目内容与技术要求

### （一）核心技术预研

1. **RISC-V 指令集深度解析**：系统研究 RISC-V 架构手册（Volume I: Unprivileged ISA），明确所选指令集版本（RV32I/RV64I）的指令编码、操作数类型、寻址方式、特权级（可选 M 模式）及异常处理机制，输出指令集功能覆盖清单。

2. **Verilog/VHDL 与工具链掌握**：熟练运用 Verilog/VHDL 语法进行硬件建模（如组合逻辑、时序逻辑、模块封装、接口定义），掌握模块化设计与层次化建模方法；搭建开发环境，配置仿真工具（ModelSim、VCS 等）、综合工具及 FPGA 开发工具链（如 Xilinx Vivado、Intel Quartus）。

### （二）CPU 架构详细设计

1. **流水线架构设计**：采用五级流水线结构，明确各阶段核心功能与模块划分，设计流水线寄存器接口与控制信号；针对冒险问题制定解决方案：数据冒险采用全旁路（Forwarding）机制，控制冒险结合分支预测与延迟槽（或冲刷机制），结构冒险通过资源分区（如分离指令/数据存储器接口）避免冲突。

2. **核心模块功能定义**：

   （1）**取指单元（IF）**：实现 PC 计数器、指令预取缓冲区（BTB 可选）、分支预测器（如两位饱和计数器），通过 Verilog/VHDL 参数化配置预取深度与预测算法；支持从指令存储器或 Cache（可选）读取指令。

   （2）**译码单元（ID）**：解析指令操作码、寄存器地址及立即数，生成控制信号（如 ALU 操作类型、访存控制）；实现寄存器堆（32 个通用寄存器），支持同步读写，通过 Verilog/VHDL 的寄存器数组与同步时序逻辑实现。

   （3）**执行单元（EX）**：包含算术逻辑单元（ALU）、移位器、比较器，支持 RISC-V 整数运算与逻辑操作；可选实现乘法器/除法器（M 扩展），通过组合逻辑优化或流水线化设计平衡性能与面积。

   （4）**访存单元（MEM）**：实现数据存储器接口，支持 Load/Store 指令的字节/半字/字访问；设计访存地址计算模块，处理数据对齐问题；可选引入数据 Cache，设计 Cache 控制器与替换算法（如 LRU）。

   （5）**写回单元（WB）**：将执行结果或访存数据写回寄存器堆，支持异常情况下的写回屏蔽；设计结果选择器，处理旁路数据与直接运算结果的优先级。

3. **参数化与可扩展性设计**：利用 Verilog 的 `parameter` 或 VHDL 的 `generic` 实现参数化设计，使 CPU 支持 RV32I/RV64I 位宽切换、流水线深度配置、Cache 大小可调；设计标准化接口（如 AXI4-Lite），方便后续扩展外设或接入片上系统（SoC）。

### （三）Verilog/VHDL 建模与模块实现

1. 按模块划分进行 Verilog/VHDL 编码，遵循硬件建模规范：组合逻辑采用 `assign` 语句（Verilog）/并行赋值语句（VHDL）或组合逻辑 `always` 块（Verilog）/进程（VHDL）实现，时序逻辑通过同步 `always` 块（Verilog）/同步进程（VHDL）与时钟信号关联，模块间通过端口定义实现接口交互；重点实现参数化寄存器堆、全旁路网络、分支预测器等核心模块。

2. 利用硬件描述语言特性优化设计：通过自定义结构体（VHDL）/用户定义类型（Verilog）定义指令类型、控制信号集，通过条件判断语句简化控制逻辑，通过模块实例化实现功能复用；编写 Verilog/VHDL 测试基准（TestBench）框架，支持模块级与系统级测试，包含激励生成、信号采集与结果比对功能。

### （四）分层仿真与验证

1. **模块级仿真**：为每个核心模块（如 ALU、寄存器堆、分支预测器）编写 TestBench，覆盖边界情况与异常场景；使用 ModelSim 进行快速功能仿真，VCS 进行高精度时序仿真，对比仿真结果与理论预期，输出模块级验证报告，确保单个模块功能正确性。

2. **系统级仿真**：搭建完整 CPU 系统仿真环境，集成指令存储器、数据存储器与测试激励模块；运行 RISC-V 官方测试集（如 riscv-tests）与自定义测试程序（覆盖所有指令类型），验证 CPU 整体功能正确性；分析流水线停顿率、分支预测准确率等关键性能指标。

3. **形式验证（可选）**：针对关键模块（如旁路网络、寄存器堆），利用形式验证工具（如 SymbiYosys、Cadence JasperGold）进行形式化验证，确保设计逻辑无死锁、无逻辑错误，提升验证覆盖率。

### （五）FPGA 原型验证

1. **Verilog/VHDL 到 FPGA 的流程适配**：直接使用 Verilog/VHDL 代码进行 FPGA 开发，配置综合选项（如模块优化等级、资源约束）；使用 FPGA 开发工具（如 Xilinx Vivado、Intel Quartus）对代码进行综合、布局布线，针对目标 FPGA 器件（如 Zynq-7000、Artix-7 系列）优化时序约束（如时钟频率、输入输出延迟）。

2. **硬件调试与测试**：将综合后的比特流文件下载到 FPGA 开发板，搭建硬件测试平台（连接串口、JTAG 调试器）；通过逻辑分析仪或 FPGA 内置 ILA 核抓取关键信号（如 PC、流水线寄存器、控制信号），调试时序问题与功能缺陷；验证 CPU 在实际硬件环境中的长期稳定运行能力。

3. **性能与资源评估**：在 FPGA 上运行 CoreMark 等基准测试程序，测试 CPU 的指令执行速度（DMIPS/MHz）、吞吐量与功耗；统计 FPGA 资源占用情况（LUT、FF、BRAM、DSP），分析设计的面积-性能-功耗平衡关系。

### （六）性能优化与应用开发

1. **性能优化实现**：基于仿真与 FPGA 测试结果，优化 CPU 架构：调整分支预测算法参数、优化旁路网络延迟、扩展指令预取缓冲区深度；可选引入一级 Cache（指令/数据分离），设计 Cache 控制器与替换策略（如 LRU、FIFO），提升访存效率。

2. **应用程序开发与演示**：开发适配该 CPU 的简单应用程序（如矩阵乘法、快速排序、LED 控制逻辑），通过 RISC-V 交叉编译器（如 riscv-gcc）编译为机器码；在 FPGA 上运行应用程序，验证 CPU 的实际应用能力，输出功能演示视频与性能分析报告。

## 六、赛程规划

为帮助参赛选手更好的备赛，企业专家提供以下赛程规划仅供参考，参赛团队可根据实际情况调整规划。

### （一）预研与架构设计

1. 完成 RISC-V ISA（RV32I/RV64I）深度调研，输出指令集功能清单；掌握 Verilog/VHDL 语法与工具链，完成开发环境搭建；编写简单模块（如 ALU、寄存器堆）验证工具链可用性。

2. 完成 CPU 架构详细设计，确定流水线结构、模块划分、冒险解决方案与参数化配置；绘制架构图、模块接口图与控制信号时序图；制定建模规范与验证计划。

### （二）Verilog/VHDL 建模与模块级验证

1. 按模块划分进行 Verilog/VHDL 编码，实现 IF、ID、EX、MEM、WB 核心模块；编写模块级 TestBench，完成组合逻辑与时序逻辑的功能仿真；修复建模错误与逻辑缺陷。

2. 集成各核心模块，搭建完整 CPU 系统的硬件模型；编写系统级 TestBench，运行 riscv-tests 基础测试用例；优化模块接口与控制逻辑，确保系统级功能正确性。

### （三）系统仿真与 FPGA 适配

1. 进行高精度时序仿真，分析流水线性能瓶颈；优化 Verilog/VHDL 代码以满足时序约束；完成与 FPGA 工具链的适配，配置初步的综合与约束参数。

2. 在 FPGA 开发工具中完成代码的综合、布局布线；配置 FPGA 引脚约束与时钟约束；下载比特流文件到开发板，完成硬件连接与基础功能调试。

### （四）性能优化与成果验收

1. 实施性能优化方案（如分支预测优化、Cache 集成）；在 FPGA 上进行性能测试与资源评估；开发应用程序，完成功能演示环境搭建。

2. 完善应用程序开发与演示；整理项目文档（设计报告、代码注释、验证报告、性能分析报告）；准备项目答辩与成果验收。

## 七、作品提交

1. **技术文档（设计文档）**：Word 版本及 PDF 版本各 1 份，建议封面或目录后的第 1-2 页项目为重要内容预览页，包含作品重要核心亮点。文档整体应包含：CPU 架构设计说明书、Verilog/VHDL 建模规范、验证计划与测试报告、FPGA 适配指南。

2. **技术数据（源代码）**：将源代码打包提交一份总的压缩包文件，内容应包括完整的 Verilog/VHDL 设计代码（含模块代码、TestBench 代码）、RISC-V 测试程序与应用程序代码，代码注释率不低于 30%。（如果为开源需在作品中明确注明）

3. **技术数据（FPGA 原型系统）**：能稳定运行的 FPGA CPU 原型，支持 RV32I/RV64I 指令集，时钟频率不低于 50MHz；输出硬件连接图、比特流文件与调试日志。

4. **技术文档（性能与验证报告）**：包含仿真与 FPGA 测试数据（指令覆盖率、分支预测准确率、DMIPS/MHz、资源占用率），对比优化前后的性能差异；附功能演示视频。

## 八、赛题难点及挑战

1. **时序收敛难题**：Verilog/VHDL 设计中，复杂模块（如旁路网络、分支预测器）易出现时序瓶颈，需掌握时序优化技巧（如寄存器重定时、资源复用、关键路径优化），确保 FPGA 上的时序收敛。

2. **流水线冒险解决的高效性**：全旁路网络与分支预测器的设计需兼顾性能与硬件开销，需通过仿真迭代优化旁路路径与预测算法参数，降低流水线停顿率，提升执行效率。

3. **模块接口兼容性**：分层建模中，各模块接口的时序与功能兼容性至关重要，需制定统一的接口规范，避免集成阶段出现交互错误。

4. **自定义设计与开源避坑**：禁止直接复用开源 RISC-V CPU（如 Rocket Chip、VexRiscv）的核心代码，需独立设计流水线与模块逻辑；可参考开源项目的设计思路，但必须进行差异化改进（如自定义分支预测器、参数化 Cache）。

## 十、参赛建议与注意事项

1. **工具链选型**：推荐使用 Verilog/VHDL+ModelSim/VCS（仿真）、Xilinx Vivado/Intel Quartus（FPGA 综合）、riscv-gcc（交叉编译）；FPGA 开发板推荐 Xilinx Zynq-7000 或 Artix-7 系列（资源充足，支持高速接口，调试工具丰富）。

2. **分阶段迭代开发**：采用"模块级验证→系统级验证→FPGA 调试"的迭代模式，每个阶段输出明确成果（如验证报告、代码版本），避免后期集中返工。

3. **文献与技术积累**：跟踪 RISC-V 基金会最新技术规范，参考硬件设计顶会论文（如 ISCA、MICRO）中的 CPU 优化方案；学习工业界 Verilog/VHDL 编码规范（如 Synopsys 设计指南），提升代码的可综合性。

4. **代码与文档规范**：Verilog/VHDL 代码遵循模块化、层次化设计原则，添加详细注释说明模块功能与设计思路；项目文档按学术论文格式撰写，明确标注设计亮点与创新点。

5. 本项目通过"架构设计→Verilog/VHDL 建模→仿真验证→FPGA 实现→性能优化"的全流程训练，将使研究生深度掌握 RISC-V 架构与硬件描述语言设计技术，具备复杂硬件系统的设计与验证能力，其成果可进一步拓展至低功耗 CPU、异构计算架构等研究方向，具有较高的学术与应用价值。
