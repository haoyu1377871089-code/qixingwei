# 项目概述

## 目标

采用 Verilog 独立设计一款 **RV32IM** 五级流水线 CPU，完成功能仿真与 FPGA 综合后仿真验证。

## 验收标准

| 项目 | 要求 |
|------|------|
| 指令集 | RV32I + M 扩展 |
| 流水线 | 5 级（IF/ID/EX/MEM/WB） |
| 冒险处理 | 全数据转发 + 分支预测 |
| 功能仿真 | riscv-tests 全部通过 |
| FPGA 综合 | Vivado 综合通过，时序收敛 ≥ 50 MHz |
| 后仿真 | 综合后网表仿真功能正确 |
| 资源 | < 5,000 LUTs |
| 目标板 | ALINX AX7020（Zynq-7020, XC7Z020CLG400-2） |

# CPU 架构

## 流水线结构

五级流水线：IF → ID → EX → MEM → WB，级间寄存器隔离。

## 模块清单

| 模块 | 文件 | 功能 |
|------|------|------|
| PC 寄存器 | `qxw_pc_reg.v` | PC 管理与下一 PC 选择 |
| 取指阶段 | `qxw_if_stage.v` | 取指 + 分支预测接口 |
| 译码阶段 | `qxw_id_stage.v` | 指令解码 + 立即数生成 + 控制信号 |
| 执行阶段 | `qxw_ex_stage.v` | ALU 运算 + 分支判断 + 地址计算 |
| 访存阶段 | `qxw_mem_stage.v` | Load/Store 数据对齐与扩展 |
| 写回阶段 | `qxw_wb_stage.v` | 结果选择与寄存器写回 |
| 寄存器堆 | `qxw_regfile.v` | 32×32bit，双读单写 |
| ALU | `qxw_alu.v` | RV32I 全部算术逻辑运算 |
| 乘除法 | `qxw_muldiv.v` | M 扩展（MUL/DIV 系列） |
| 数据转发 | `qxw_forwarding.v` | EX/MEM/WB 三路旁路转发 |
| 冒险控制 | `qxw_hazard_ctrl.v` | stall/flush 控制 |
| 分支预测 | `qxw_branch_pred.v` | 两位饱和计数器 BHT（256 项） |
| CSR | `qxw_csr.v` | M-mode 基础 CSR |
| CPU 顶层 | `qxw_cpu_top.v` | 核心顶层封装 |

## 冒险处理

- **数据冒险**：EX/MEM/WB 全转发，Load-Use 插入 1 个 bubble
- **控制冒险**：BHT 分支预测，错误时冲刷 IF/ID
- **结构冒险**：指令/数据存储器分离（Harvard 架构）

## SoC 最小系统

| 组件 | 地址 | 大小 | 说明 |
|------|------|------|------|
| 指令 ROM | 0x0000_0000 | 16 KB | BRAM |
| 数据 RAM | 0x0001_0000 | 16 KB | BRAM |
| UART | 0x1000_0000 | 256 B | 串口输出（调试） |
| Timer | 0x1000_1000 | 256 B | 64-bit 计时器 |

# 关键技术要点

## FPGA 综合注意事项

1. 所有 RTL 需可综合：禁用 `$display`、`initial`（仅限仿真）、`#delay` 等不可综合语法
2. 乘法器映射到 DSP48E1 slice，避免占用 LUT
3. 存储器推断为 BRAM，避免占用分布式 RAM
4. 异步复位统一转同步复位，防止亚稳态
5. 所有模块时钟域统一，无跨时钟域

## 后仿真要点

1. **综合后功能仿真**（Post-Synthesis Functional Simulation）：验证综合优化未改变逻辑功能
2. **布线后时序仿真**（Post-Implementation Timing Simulation）：加载 SDF 延时文件，验证实际时序下功能正确
3. 后仿真需使用 Vivado 仿真器或兼容 SDF 的仿真器
